## 前言

### 开源从何入手？

尽管自己有用了两三年的 GitHub 了，平时也一直有在关注各种开源的资讯；但是我从来没有真正参与到一个开源项目中，而开源精神是我一直向往的。

但是，我不知道该选什么项目去参与。

开源项目有非常多，其中肯定也有很大一部分是和我擅长的 Web 技术栈相符合的；而我却一直不知道该从何着手，更别说具体应该如何去贡献代码了。所以最根本的原因，我就是缺乏一个参与的动机。

### 契机

我一直很想做个人网站，前前后后研究了有个半年的时间了；但是由于不知道如何在网页中去组织关于自己的信息，再加上缺乏设计方面的灵感，就一直没进展。

前段时间我还是憋不住了，想着就算做出来不好看，还是得建一个个人信息的网页入口，总有一天会用到。于是用自己相对熟悉的 HeroUI 开始手搓。

过了一两周做出了一个半成品之后，感觉还可以；于是我把网站进行了部署。有一次在用手机访问的时候，我发现由于自己没有去做响应式布局（很耗时），网站在小屏幕上的效果惨不忍睹。与此同时，我觉得自己做的主页一直看着很别扭，但是又没有时间和耐心去专门设计；于是我终于妥协了，想去找一个现成的模版套。因为我的当务之急是呈现内容，而不是把页面做得独特，因为这要花很长时间。

### Magic Portfolio

就这样，我在 Vercel 官网推荐的 Portfolio 模版中，发现了一个让我眼前一亮的项目：来自 once-ui 的 Magic Portfolio。点进预览网站一看，真的太喜欢了：响应式布局、简洁而不失优雅的设计，完全长在了我的审美上。

## 渐进式参与

### 初步参与

在开发过程中，我发现的第一个问题，就是在博客界面中，TOC 会出现重复的聚焦元素；研究了之后发现是中文所导致的错误。

一开始，我的想法是在自己的仓库中修复一下就好了；但是转念一想，这个修复其实对于所有非英文的情况都适用的。而这个项目如果在今后在全球变得更加普及，那么这个问题肯定是会被母语非英文的开发者所遇到的。

于是我就萌生了提 PR 的想法；但随后忽然意识到，自己这个行为其实就是参与到了开源中！剖析了一下心理之后，我认为自己是一开始被项目所吸引，惊叹自己可以轻松地使用到这么高质量的资源，对仓库 Owner 的敬佩油然而生；于是认为这么好的项目肯定有很好的前景，应当也必然会被更多人接触到。

就这样不经意间，我和 once-ui 产生了这种 bonding；让我觉得参与到这个项目中是一件很有使命感，同时又值得我去付出贡献的事情。换言之，我对这个项目产生了一种归属感。

提 PR 之前，我决定还是先提一个 issue，看看 Owner 的态度。

[Slugify removes non-Latin characters from blog headings, causing duplicate IDs in TOC](https://github.com/once-ui-system/magic-portfolio/issues/130)

### 进一步参与

第二天，我就收到了 Owner 及时的回复；能看得出来，他对于贡献的态度还是很鼓励的。

![](/images/blogs/once-ui-getting-involved/image-01.png)

于是我就开始提 PR。虽然这个 issue 是一个很简单的 fix，但是由于是自己第一次给别人的大仓库提 PR，还是比较束手束脚，把各种 edge case 都测了一遍，又检查了好几次，才最终把代码提交。

### 融入社区

就这样，在之后发现了问题之后，我就果断提 issue。除此之外，我还加入了 once-ui 的 Discord 社群，抛出自己遇到的问题，和来自全球各个地方的开发者共同讨论。大家的态度也都很好，在别人分享的时候会鼓励，在有人遇到问题的时候会积极给出解决方案。

[加入 once-ui 的 Discord 社区](https://discord.com/invite/5EyAQ4eNdS)

## 我的感想

从一个开源文化的消费者，转变到一个实际的贡献者角色，只花了我一个周末的时间。虽然只是初出茅庐，但是通过这次的经历，是真切地感受到了开源社区良好的氛围；大家都不约而同地向着一个相同的目标前进，这是让我很感动的一个点。

我认为 once-ui 还有很大的成长空间，而我希望自己能够为这个项目做出更多的贡献。这个过程不仅能学习到技术，更重要的是能和社区一起成长。
